# Marketing — спецификация MVP

Личное приложение для маркетолога. Single-user, локальное хранение, без бэкенда и БД на старте.

## Цель MVP

Закрыть цикл «увидеть → спланировать → создать»:

1. Видеть свои метрики по каналам (Дашборд)
2. Планировать контент в едином календаре (Контент-план)
3. Быстро генерировать тексты, идеи, брифы (AI-ассистент)

## Контекст

- **Каналы:** РФ-стек — VK, VK Ads, Telegram, Яндекс.Директ, Яндекс.Метрика
- **Источник данных:** ручной ввод + CSV-импорт (без OAuth-интеграций на старте)
- **Пользователи:** только владелец, без авторизации
- **Стек:** Next.js 14 App Router, TypeScript, Tailwind, без БД (localStorage)
- **AI:** Anthropic Claude API (ключ в env, опционально)

## Маршруты (App Router)

```
/                 → редирект на /dashboard
/dashboard        → KPI-карточки + фильтры + таблица метрик
/content          → канбан контент-плана + форма поста
/ai               → AI-ассистент с пресетами
/settings         → API-ключ, каналы, экспорт/импорт JSON
/api/ai/generate  → серверный route, проксирует запрос к Claude
```

## Модели данных

См. `lib/types.ts`.

## Хранилище

Всё в `localStorage` под ключами:

- `marketing.channels`
- `marketing.metrics`
- `marketing.posts`
- `marketing.settings`

Экспорт/импорт JSON в Настройках — для переезда между машинами.

## Что НЕ входит в MVP

- OAuth-интеграции с площадками
- Командная работа, роли, проекты клиентов
- Медиаплан с бюджетами (вторая итерация)
- Конкуренты (третья)
- Отчётность с PDF-генерацией (третья)

## Дальнейшие итерации

1. **v2:** медиаплан + ROI, базовая аналитика трендов (неделя/неделя)
2. **v3:** автоматический pull из Яндекс.Метрики (есть открытое API)
3. **v4:** конкуренты (ручной ввод + сравнительные графики)
4. **v5:** генератор отчёта в PDF/презентацию
